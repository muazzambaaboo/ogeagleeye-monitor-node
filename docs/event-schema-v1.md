# Event Schema v1 — Canonical Wire Contract

> **Frozen at end of Loop 2.** Schema changes require a new API version (`/api/v2`).

## Endpoint

| Item | Value |
|------|-------|
| Method / path | `POST /api/v1/events` |
| Auth header | `X-OGEagleEye-Key: <project_ingest_key>` |
| Content-Type | `application/json` |
| Max body | **200 KB** |
| Success response | `202 {"accepted":true}` |
| Duplicate response | `202 {"accepted":true,"duplicate":true}` |

## Middleware (ingestion only)

Ingestion routes share **zero** middleware with the Filament panel (no session, no CSRF, no Livewire).

Pipeline:

1. Reject oversized body → `413`
2. Authenticate ingest key → `401` if missing/unknown (project lookup cached 60s)
3. Rate limit → `429` + `Retry-After` (default **600 events/min/project**; override via `projects.settings.rate_limit_per_minute`)
4. Validate schema → `422` with machine-readable `errors[]`
5. Idempotency (`Cache::add` / Redis SETNX equivalent) → duplicate short-circuit
6. `ProcessEventJob` on the `ingest` queue → `202`

## Rules

| Rule | Detail |
|------|--------|
| Idempotency | `event_id` is the client idempotency key. Server stores `ingest:evt:{project_id}:{event_id}` for **24h**. Duplicates return `202` with `"duplicate":true` and do **not** re-queue. |
| Conditional fields | `exception` **required** when `event_type=error`. `request` **required** when `event_type=http_failure`. |
| Timestamps | Server **never** trusts `timestamp` for ordering. Workers/DB also record `received_at`. |
| PII | SDKs hash IP/email by default; raw values only if the project setting enables it. |
| Context size | `context` JSON must be ≤ **8 KB**. |

## Error responses

| Status | When | Body shape |
|--------|------|------------|
| `401` | Missing or unknown `X-OGEagleEye-Key` | `{ "message": "..." }` |
| `413` | Body > 200 KB | `{ "message": "...", "errors": [{ "code": "payload_too_large", ... }] }` |
| `422` | Schema validation failed | `{ "message": "...", "errors": [{ "code", "field", "message" }, ...] }` |
| `429` | Rate limit exceeded | `{ "message": "..." }` + header `Retry-After` |

## `event_type` values

- `error` — uncaught exception / captured error
- `http_failure` — failing or slow HTTP response
- `heartbeat` — liveness ping
- `bug_report` — user-submitted bug report
- `scan_result` — malware / integrity scan findings (see [`scanning.md`](scanning.md); context.scan carries findings)
- `log` — structured log event
